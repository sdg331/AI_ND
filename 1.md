네, 지금까지 구현된 **FridgeChef** 프로젝트의 기술 명세서(Technical Specification)를 정리해 드립니다. 학교 과제 보고서나 발표 자료에 바로 사용하실 수 있도록 전문적인 용어로 정리했습니다.

-----

### 1\. 데이터셋 학습 여부 (Dataset Training)

* **학습 여부:** 별도의 커스텀 데이터셋을 사용하여 모델을 **직접 학습(Fine-tuning)시키지 않았습니다.**
* **사용 방식:** Google에서 이미 방대한 데이터로 학습시킨 \*\*사전 학습된 모델(Pre-trained Model)\*\*을 사용했습니다.
* **기술적 설명:**
* 대신 **프롬프트 엔지니어링(Prompt Engineering)** 기법을 사용하여, 사용자의 냉장고 데이터(Google Sheets)와 프로필 정보를 프롬프트(질문)에 포함해 AI에게 전달하는 **인컨텍스트 러닝(In-context Learning)** 방식을 적용했습니다.
* 즉, AI를 '가르친' 것이 아니라, AI에게 현재 상황(냉장고 재료)을 '설명'해주고 답을 얻는 방식입니다.

### 2\. 사용 모델 (AI Models)

이 프로젝트는 **Google Gemini Family** 모델을 상황에 맞춰 하이브리드로 사용합니다. (코드상 자동 전환 로직 적용)

| 기능 | 사용 모델 (Model Name) | 역할 및 특징 |
| :--- | :--- | :--- |
| **영수증/사진 등록** | **Gemini 1.5 Flash**<br>(Fallback: Gemini Pro Vision) | **멀티모달(Multimodal)** 모델. 이미지와 텍스트를 동시에 이해하여 식재료 객체를 인식하고 JSON 데이터로 변환합니다. |
| **레시피 생성** | **Gemini 1.5 Flash**<br>(Fallback: Gemini 1.0 Pro) | **텍스트 생성(Text Generation)** 모델. 빠른 응답 속도와 긴 컨텍스트 처리가 가능한 모델을 사용하여 재료 조합 및 조리법을 생성합니다. |
| **챗봇 (쿠킹 어시스턴트)** | **Gemini 1.5 Flash** | 사용자의 질문 의도를 파악하고 자연스러운 대화형 답변을 제공합니다. |

### 3\. 인증 토큰 및 보안 (Authentication & Security)

본 프로젝트는 두 가지 종류의 인증 방식을 사용합니다.

* **Google Gemini API:**
* **방식:** **API Key** 인증
* **저장소:** 환경 변수 파일(`.env`) 내 `GEMINI_API_KEY`
* **역할:** AI 모델 사용 권한을 인증하고 사용량(Quota)을 관리합니다.
* **Google Sheets API:**
* **방식:** **Service Account (서비스 계정)** 기반의 **OAuth 2.0** 인증
* **저장소:** `backend/service_account.json` (JSON 키 파일)
* **역할:** 서버가 사용자 대신 Google Cloud 프로젝트의 권한으로 스프레드시트에 접근하여 데이터를 읽고 쓰기(RW) 위한 서버 간 인증입니다.

### 4\. API (Application Programming Interfaces)

#### (1) 외부 API (External APIs)

프로젝트가 의존하고 있는 외부 서비스입니다.

* **Google Generative Language API (REST):** Python의 `requests` 라이브러리를 통해 직접 HTTP 요청을 보내 AI 기능을 수행합니다.
* **Google Sheets API / Drive API:** `gspread` 라이브러리를 통해 데이터베이스 역할을 하는 스프레드시트와 통신합니다.

#### (2) 내부 API (Internal Backend APIs)

우리가 `app.py` (Flask)에서 직접 만든 API 엔드포인트입니다.

| HTTP Method | 엔드포인트 (Endpoint) | 기능 설명 |
| :--- | :--- | :--- |
| **GET** | `/ingredients/<user_id>` | 특정 사용자의 냉장고 재료 목록 조회 |
| **POST** | `/ingredients/<user_id>` | 텍스트로 재료 직접 추가 (중복 시 합침) |
| **POST** | `/ingredients/vision` | 사진/영수증을 분석하여 재료 자동 추가 |
| **POST** | `/ingredients/delete/<id>` | 재료 삭제 |
| **POST** | `/ai/generate` | 현재 재료와 프로필을 기반으로 맞춤 레시피 생성 |
| **POST** | `/users/<user_id>` | 사용자 프로필(알레르기, 도구 등) 저장 |
| **GET/POST** | `/community/posts` | 커뮤니티 게시글 조회 및 작성 |
