# Pre 기획안 - cookmake (내 냉장고 속 AI 셰프)

---

## 1. 프로젝트 개요 (Project Overview)

- 프로젝트명: cookmake (내 냉장고 속 AI 셰프)
- 슬로건: 냉장고 속 재료로 완성하는 쉽고 스마트한 집밥
- 서비스 한 줄 정의: Gemini AI가 냉장고 속 재료와 개인 프로필을 분석해 직접 레시피를 생성하고, 요리 과정 중 문제 해결까지 도와주는 맞춤형 AI 쿠킹 도우미 앱

### 문제 정의

- 많은 1인 가구가 냉장고 속 재료를 활용하지 못해 음식물 낭비로 이어짐  
- 배달 의존도가 높고, 요리 초보자는 복잡한 레시피 검색 과정에 피로감을 느낌  
- 기존 레시피 앱은 개인의 상황(도구, 수준, 취향)을 제대로 반영하지 못함  

### 솔루션

- 사용자는 사진 업로드 또는 텍스트 입력으로 재료를 손쉽게 “나의 냉장고”에 등록  
- Gemini Vision이 재료 인식 후 Google Sheets에 저장하고,  
  Flask 백엔드가 재료·프로필·기록을 통합 관리  
- Gemini Flash가 입력된 정보로 “실시간 개인 맞춤형 레시피”를 자동 생성  
- AI 요리 챗봇이 조리 중 발생하는 문제를 실시간으로 해결  
- 커뮤니티 없이도 개인의 성취감과 효율성을 중심으로 동기부여 강화

## 2. 시장 분석 (Market Analysis)

- 타겟 시장: AI 개인화 푸드테크, 20~30대 1인 생활가구  
- 시장 규모 및 성장성:
  - 글로벌 푸드테크 시장은 2024년 기준 약 2,200억 달러 규모로 추산  
  - AI 개인화 푸드 및 헬스케어 시장은 향후 10% 이상 성장 예상  
- 소비자 트렌드:
  1. 개인화된 AI 서비스 선호 증가 (특히 MZ세대 중심)
  2. 비용 절감 + 지속가능성, 음식 낭비 절감에 대한 사회적 관심 증대
  3. 집밥을 하나의 경험 콘텐츠로 인식하는 경향 강화  
- 시장 기회:
  - AI 이미지 인식 + LLM API(무료 티어) 접근성 향상  
  - 음식물 쓰레기 절감 트렌드와 ESG 연결성  
  - 단순·저비용 모바일 MVP로 빠른 시장 검증 가능

## 3. 핵심 기능 상세 (Detailed Features)

### 3.1. MVP 핵심 기능

- 나의 냉장고  
  - 텍스트 입력 또는 재료 사진 업로드  
  - Gemini Vision을 활용해 재료명 자동 인식  
  - Google Sheets DB에 재료를 자동 저장  
- AI 셰프 (레시피 생성)
  - Flask 백엔드가 Gemini Flash를 호출하여 맞춤형 레시피 생성  
  - 예상 조리 시간, 난이도, 단계별 가이드 포함  
- AI 쿠킹 어시스턴트
  - “간이 짜요”, “대체 재료 뭐 써요?” 등 AI 챗봇 질의응답 기능  

### 3.2. 확장 기능 (Full Features)

- 유통기한 자동 관리 및 “긴급 요리 추천”  
- 개인 프로필 기반 필터링 (비선호/알레르기 반영)  
- 단순 공유 기능(Phase 2: Cook Log History UI 중심)  

---

## 4. 기술 스택 (Technology Stack)

- 프론트엔드: React Native (Expo)
- 백엔드: Python Flask REST API
- 데이터베이스: Google Sheets (gspread 라이브러리)
- AI 모델:
  - Gemini Vision (재료 인식)
  - Gemini Flash (레시피 생성 및 요리 챗봇)
- 호스팅: Render / Railway / PythonAnywhere (무료 플랜)
- 인증·보안: dotenv 환경변수, 기본 Bearer Token, CORS 관리  

---

## 5. 수익 모델 (Business Model)

- 기본 무료 (모든 기능 사용 가능, 일일 호출 제한만 존재)
- 프리미엄 버전 (예정)
  - 추가 Gemini 호출, 요리 로그 자동 분석 제공  
- 제휴 및 간접 수익
  - 푸드 브랜드 협력 → 재료 구매 유도형 광고 삽입
  - 채용/쿠킹 클래스 연계형 커뮤니티 이벤트  

---

## 6. 성공 지표 (Success Metrics)

- MAU(월간 활성 사용자수)  
- AI 레시피 생성 횟수 / 사용자  
- 챗봇 기능 사용률  
- 첫 요리 성공률(재시도율) 70% 이상  
- p95 Gemini API 응답 시간 ≤ 3초  
- Google Sheets 동기화 오류율 ≤ 1%  

---

## 7. 타겟 정의 (Target Audience)

- 1차 타겟: 대학생, 사회초년생 등 요리 초보의 1인 가구  
  - 요리 시간 및 비용 절약에 관심  
- 2차 타겟: 건강식/비건/다이어트 등 특정 식단을 원하는 사용자  
  - 재료와 목표에 맞춘 식단 관리 니즈  

---

## 8. 사용자 흐름 (User Flow)

1. 회원가입 및 프로필 생성  
   - 선호 음식, 보유 도구, 요리 수준 정보 입력  
2. 재료 등록  
   - 냉장고 재료를 텍스트 입력 또는 사진 업로드  
   - Gemini Vision이 자동 인식 → Flask로 전송 → Google Sheets 저장  
3. 레시피 생성  
   - 사용자는 버튼 한 번으로 레시피 요청  
   - Flask 서버가 Gemini Flash API 호출, 3초 내 응답  
4. 요리 중 챗봇 대화  
   - 사용자의 요리 관련 질문에 실시간 응답  
5. 로그 저장  
   - Google Sheets에 사용 내역(요청, 레시피, 챗봇 로그) 기록

## 사용자 흐름 (예시)

1. 앱 실행 → 간단한 온보딩  
2. “나의 냉장고”에서 사진 업로드 (예: 양파, 두부, 달걀)  
3. 인식된 재료 목록 자동 표시  
4. “레시피 만들기” 클릭 → Gemini Flash 결과: “두부덮밥 추천”  
5. 요리 중 “밸런스가 이상해요” → 챗봇: “간을 조금 줄이고 물을 추가하세요” 응답  
6. 요리가 끝나면 로그 저장 + 다음 요리 제안

## 출시 계획 (Release Plan)

- Phase 1: MVP (기본 기능)
  - 재료 등록 / 레시피 생성 / 챗봇  
  - 목적: AI 기반 레시피 가치 검증
- Phase 2: 고도화
  - 유통기한, 커뮤니티, 챌린지 기능 추가

## 차별성 (Competitiveness)

- 완전 무료: Gemini API 무료 티어 최대 활용, Google Sheets 기반 DB  
- 초개인화: 개인 프로필 + 보유재료 + 도구까지 고려  
- 실시간 대응형 챗봇: 요리 과정 전반에서 AI가 코치 역할 수행  
- 단순·가벼운 구조: 1인 개발자가 유지보수 가능한 규모

## 기대 효과 (Expected Outcome)

- 사용자 가치: 재료 낭비 최소화, 요리 자신감 향상  
- 사회적 가치: 음식물 쓰레기 및 배달 의존도 감소  
- 비즈니스 가치: 구독 기반 확장성, 브랜드 협업 가능성  
- 기술적 가치: Gemini API를 활용한 실시간 개인화 서비스 사례 확보
