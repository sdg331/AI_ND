# cookmake PRE 기획안
## 내 냉장고 속 AI 셰프

---

## 1. 프로젝트 개요

### 프로젝트명
**cookmake** (내 냉장고 속 AI 셰프)

### 슬로건
냉장고 재료로 만드는 AI 요리 도우미, 식비 절감과 음식 낭비 제로

### 문제 정의

20~30대 1인 가구 및 자취생은 냉장고 속 재료를 어떻게 활용할지 불명확해 결국 음식물 낭비로 이어지는 경우가 많습니다. 또한 레시피 검색은 번거롭고, 요리 과정 중 어려움에 직면할 때마다 인터넷을 다시 검색해야 하는 비효율이 발생합니다. 기존 레시피 앱은 일방적 정보 제공만 하고, 실시간 요리 보조 및 개인 맞춤화가 부족합니다.

### 솔루션

사용자가 냉장고 속 재료를 **사진 촬영 또는 텍스트 입력**하면, **Google Gemini Vision AI가 재료를 인식·분석**한 후 **사용자의 알레르기·선호도·조리 난이도에 맞춘 맞춤형 레시피**를 추천합니다. 더 나아가 요리 진행 중 **실시간 AI 챗봇**이 단계별 가이드와 문제 해결 조언을 제공하여 요리 완성율을 높입니다.

### 목표 시장

자취생, 1~2인 가구 직장인, 요리 초보자, 식비 절감 및 건강 식단 관심층  
(주요: 20~30대, 서울·수도권)

### 서비스 한 줄 정의

냉장고 재료 사진을 찍거나 입력하면 AI가 실시간 레시피와 요리 가이드를 제공하는  
**"식비 절감 + 요리 성공률 극대화"** 모바일 요리 도우미 서비스

## 2. 시장 분석 (Market Analysis)

cookmake는 **푸드테크 시장 + AI 개인화 서비스 시장 + 1인 가구 식문화 시장**을 동시에 공략합니다.

### 글로벌 푸드테크 시장 기회

- **글로벌 푸드테크 시장:** 2023년 약 2,200억 달러 규모, 2028년까지 연평균 성장률(CAGR) 9% 이상 전망
- **AI 기반 음식 추천 서비스:** 2022~2027년 연평균 25% 성장 중 (음성 인식, 이미지 분석 기술 고도화)

### 국내 시장 기회

- **HMR(가정간편식), 밀키트, 신선 배송 시장:** 약 7조 원 규모, 연평균 15% 성장률
- **1인 가구 증가:** 전국 800만 가구(약 25%), 서울·수도권 35% 이상
- **음식물 쓰레기 문제:** 국내 연간 약 570만 톤 발생, 식품 폐기 절감 사회적 관심 증가

### 소비자 트렌드

#### 자취/소규모 가구 확산
1인 가구 비율 연 2~3% 증가 → 소량 재료 활용 서비스 수요 급증

#### 건강·환경 친화적 소비 확산
MZ세대의 "제로 웨이스트", "식품 낭비 제로" 관심도 높음

#### AI 개인화 서비스 기대
일방적 검색이 아닌 **"나에게 맞춘 추천"** 선호 (ChatGPT, 음악 추천 알고리즘 영향)

#### 모바일 우선 문화
인스턴트 요리 정보 수요 (동영상 레시피, SNS 공유 중심)

### 경쟁 기회 요인

| 요인 | 설명 |
|------|------|
| **음식물 쓰레기** | 전 세계 연 13억 톤 이상, 개인 가구의 "재료 활용 효율화" 니즈 명확 |
| **AI 이미지 인식 고도화** | Google Gemini, GPT-4V 등 무료/저비용 API 대중화로 구현 장벽 낮아짐 |
| **소셜 공유 문화** | 사용자가 만든 요리 사진을 인스타그램, 틱톡에 자연스럽게 공유 (자발적 마케팅) |

## 3. 핵심 기능 (Core Features)

### MVP (Minimum Viable Product) — Phase 1 (출시 기본 기능)

#### 1. 재료 인식 (멀티모드)

- **사진 업로드:** 냉장고 사진 → Google Gemini Vision으로 재료 자동 인식
- **텍스트 입력:** 사용자가 "계란, 감자, 양파, 버터" 직접 입력
- **결과 수정 UI:** 인식 오류 시 사용자가 재료 추가/삭제 가능

#### 2. AI 레시피 추천

- **인식된 재료 기반 추천:** Google Gemini로 "이 재료로 만들 수 있는 요리 5가지" 생성
- **사용자 프로필 필터:** 알레르기, 선호 식단(채식/저탄고지/일반), 조리 난이도(초보/중급/고급), 조리 시간(15분/30분/60분+)
- **카드형 UI:** 각 레시피는 제목, 주요 재료, 조리 시간, 난이도, 이미지 포함

#### 3. 단계별 요리 가이드

- **선택한 레시피 상세 보기:** 필요한 재료, 단계별 조리법 (1단계: 양파 다지기 → 2단계: 기름에 볶기 등)
- **영양 정보:** 칼로리, 단백질, 탄수화물, 지방 (Gemini로 추정)

#### 4. 실시간 AI 요리 챗봇 (핵심 차별화 기능)

- **즉시 질문 대응:** 요리 진행 중 "계란이 자꾸 굳어요", "양파가 너무 많아요" 같은 질문에 챗봇이 대답
- **단계별 팁:** "이 단계에서는 약불로 3분 정도 할게요"
- **문제 해결:** "계란이 굳으면 찬물에 넣고 몇 초 정도만"

#### 5. 사용자 프로필 설정

- **기본 정보:** 이름, 나이/성별(선택), 거주 지역
- **요리 취향:** 선호 음식(한식/양식/중식 등), 피하는 재료(알레르기/채식/건강상 이유)
- **조리 도구:** 있는 도구 선택(냄비, 전자레인지, 오일, 버터 등) — 조리법 추천에 반영
- **난이도:** 조리 경험 수준(초보/중급/고급)

### 추가 고려 기능 (Phase 2~3 — 초기 출시 이후 확장)

- 난이도·시간별 필터: "15분 이내 간단 요리", "초보자용 밥 요리"
- 영양 관리: 칼로리 목표 설정 후 추천 레시피의 영양 정보 비교
- 레시피 저장/즐겨찾기: 마음에 드는 레시피 모음
- 가족/공유 기능: 여러 프로필 설정 (가족 구성원별 추천)
- 요리 기록: "오늘 만든 요리" 기록 및 SNS 공유 버튼
- 장보기 연동: 부족한 재료를 쿠팡, 마켓컬리 장바구니로 자동 추가 (향후 제휴)

## 4. 기술 스택 (Technology Stack)

**초저비용 원칙 준수** — 무료/저비용 오픈소스 및 무료 API 우선 활용

| 계층 | 기술 | 선택 이유 |
|------|------|----------|
| **프런트엔드** | React Native (Expo) | iOS/Android 동시 지원, 빌드 비용 최소화, 핫 리로드 개발 효율 |
| **백엔드** | Python Flask | 가벼움, 빠른 프로토타이핑, 무료, 학생 친화적 |
| **데이터베이스** | Google Sheets + gspread | 완전 무료 DB 역할, 초기 CRUD 비용 0원 (향후 규모 커지면 PostgreSQL 전환) |
| **AI 이미지 인식** | Google Gemini Vision (무료 티어) | 무료 API, 한국어 지원, 식품 인식 성능 우수 |
| **AI 텍스트 생성** | Google Gemini Flash (무료 티어) | 무료 API, 빠른 응답, 레시피 생성/챗봇 용도 최적 |
| **오프라인 폴백** | TensorFlow Lite (선택) | 로컬 추론, 네트워크 없을 때 기본 재료 인식 (완전 선택) |
| **호스팅 (백엔드)** | Render / Railway / PythonAnywhere (무료 티어) | 무료 배포, 슬립 제한 있지만 소규모 수용 가능 |
| **이미지 저장** | Google Cloud Storage 무료 티어 OR AWS S3 (프리 티어) | 초기 무료, 저비용 확장성 |
| **인증/토큰** | JWT (간단 구현) | 무료, 경량 (Firebase Auth 선택 시 무료 제한 있음) |

### 아키텍처 다이어그램 (개념)

```
사용자 앱 (React Native/Expo)
    ↓ (API 호출: Axios/fetch)
Python Flask 백엔드 (Render/Railway)
    ├─ /recipe (레시피 생성) → Google Gemini Flash API
    ├─ /chat (요리 챗봇) → Google Gemini Flash API
    ├─ /ingredients (재료 인식) → Google Gemini Vision API
    ├─ /profile (사용자 프로필) → Google Sheets (gspread)
    └─ 로깅/분석 → Google Sheets
```

## 5. 수익 모델 (Business Model)

### Phase 1 (초기 6개월): 무료 모든 기능 + 데이터 수집

- **목표:** 사용자 확보, 서비스 안정화, 피드백 수집
- **수익:** 없음 (초기 투자 최소화 중심)

### Phase 2 (6개월~1년): 프리미엄 모델 도입

| 모델 | 세부 내용 | 예상 수익 |
|------|----------|----------|
| **프리미엄 구독** | 월 4,900원: 광고 제거, 레시피 무제한 저장, AI 챗봇 우선순위 처리, 영양 분석 고급 기능 | 10,000 구독자 기준 월 4,900만 원 |
| **광고/제휴** | 식품 브랜드(CJ, 농심), 온라인 마켓(쿠팡, 마켓컬리) 광고 배너 또는 스폰서 레시피 | 월 500만~1,000만 원 (초기) |
| **커머스 연동** | 부족한 재료 자동 장바구니 연결 → 수수료 3~5% | 월 500만~2,000만 원 (향후) |

### 초기 비용 구조 (월)

- Google Gemini API: 무료 티어 (월 60회 호출 제한, 추가 비용 ~$0.075/1K 토큰)
- Render/Railway 호스팅: $5~7 (무료 티어도 가능, 슬립 있음)
- Google Sheets + Cloud Storage: 무료 (초기 용량 15GB 무료)
- **총 월 비용: 5,000~10,000원 (거의 무료 수준)**

### 초기 목표 (3개월)

- 앱 다운로드: 5,000명 이상
- 월 활성 사용자(MAU): 1,000명 이상
- 일일 활성 사용자(DAU): 200명 이상
- 일일 레시피 생성 건수: 300건 이상

### 중기 목표 (6개월)

- 다운로드: 50,000명
- MAU: 10,000명
- DAU: 2,000명
- 평균 재방문율(주 1회 이상 사용): 30% 이상
- AI 챗봇 평균 만족도: 4.0/5.0 이상

### 주요 지표

| 지표 | 설명 | 측정 방식 |
|------|------|----------|

| **이탈율** | 월 단위 미사용자 비율 | (이전월 - 금월 사용자) / 이전월 사용자 |

## 7. 타겟 정의

### 1차 타겟: 요리 초보 자취생 및 1~2인 직장인 가구

| 항목 | 내용 |
|------|------|
| **주요 소비층** | 대학생(20~25세), 사회초년생(25~30세), 직장인(30대 초반) |
| **지역** | 서울, 경기, 인천 (수도권 80% 우선) |
| **특징** | 외로움/바쁨 → 간편식 선호, 형편 없음 → 식비 절감 관심 |
| **주요 니즈** | 냉장고 재료를 쉽게 활용, 간단한 요리 아이디어 빠르게 얻기, 요리 실패율 낮추기 |
| **구매력** | 월 식비 20~30만 원, 외식 대체 수요 |
| **접근 채널** | 인스타그램, 틱톡, 대학 커뮤니티(에브리타임) |

### 2차 타겟: 건강 관리·다이어트 관심층

| 항목 | 내용 |
|------|------|
| **주요 소비층** | 직장인(28~40세), 헬스 관심층, 영양 관리 실천층 |
| **주요 니즈** | 칼로리 관리, 고단백 식단, 다이어트 레시피 추천, 영양소 추적 |
| **니즈 충족 방식** | 프리미엄 구독(영양 분석 고급 기능) |
| **접근 채널** | 헬스/피트니스 커뮤니티(식단 카페, 헬스 인플루언서) |

### 3차 타겟 (향후 확장): 가족 단위 소비자

- 자녀 영양 관리, 식품 낭비 절감 관심
- 여러 가족 구성원 프로필 지원으로 가치 확대
