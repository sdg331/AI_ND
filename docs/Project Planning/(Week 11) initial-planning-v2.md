## 1. 프로젝트 개요

**cookmate (내 냉장고 속 AI 셰프)**는 AI가 사용자의 냉장고 속 재료와 개인 프로필을 분석해 최적의 맞춤형 레시피를 생성하고, 요리 과정의 문제 해결까지 실시간으로 지원하는 초개인화 쿠킹 솔루션입니다.

---

## 2. 문제 정의 및 솔루션

### 2.1 핵심 문제

#### 재료 활용의 비효율
냉장고에 재료가 있어도 활용법을 몰라 유통기한이 지나 버려지는 음식물 낭비가 발생합니다.

#### 레시피 탐색의 피로감
요리 초보자나 1인 가구가 자신의 상황(보유 재료, 조리도구, 시간)에 딱 맞는 레시피를 찾는 것이 번거롭습니다.

#### 요리 중단
요리 과정에서 발생하는 돌발 상황(예: 간이 짜다, 재료가 부족하다)에 실시간으로 대처하기 어렵습니다.

### 2.2 솔루션 가설

**"AI가 냉장고 재료와 개인 프로필을 분석하여 맞춤형 레시피를 제공하고, 요리 과정을 실시간 챗봇으로 지원한다면, 음식물 낭비를 줄이고 요리 초보자의 요리 경험을 획기적으로 향상시킬 수 있을 것이다."**

## 3. 핵심 기능

### (1) 나의 냉장고 (간편한 재료 등록)

간편한 식재료 등록을 위한 멀티모드 입력:

- **사진 촬영:** 냉장고 내부를 촬영하면 Google Gemini Vision으로 객체 인식하여 자동 등록
- **영수증 스캔:** OCR(광학 문자 인식)을 통해 구입한 식재료 일괄 등록
- **텍스트 입력:** 사용자가 직접 재료명 입력
- **등록된 재료 목록:** 추가한 모든 재료를 카드형 UI로 시각화하여 관리

### (2) 개인 프로필

사용자의 취향과 상황을 종합적으로 반영하는 프로필 설정:

- **선호/비선호 재료:** 좋아하거나 싫어하는 음식 또는 재료 지정 (예: "버섯 싫어요", "고기 좋아해요")
- **알레르기 정보:** 알레르기 유무 및 종류 입력 (예: 갑각류, 땅콩, 계란)
- **보유 조리도구:** 사용 가능한 주방 기기 선택 (예: 에어프라이어, 오븐, 믹서기, 전자레인지)
- **개인 목표:** 식단 목표 설정 (예: 다이어트, 비건, 저염식, 고단백)
- **선호 조리 난이도:** 조리 경험 수준 선택 (초보/중급/고급)

### (3) AI 셰프 (레시피 생성 및 조리 지원)

사용자 맞춤형 요리 경험을 제공하는 AI 기반 주요 기능:

- **맞춤 레시피 생성:** 등록된 재료와 개인 프로필을 종합 분석하여 최적의 맞춤 레시피 생성 및 추천
  - 보유 재료 기반으로 만들 수 있는 요리 추천
  - 개인 목표(다이어트/비건 등)와 조리도구를 고려한 필터링
  - 조리 시간, 난이도별 정렬 옵션

- **단계별 조리 가이드:** 선택한 레시피의 상세 조리법 제공
  - 1단계, 2단계, 3단계 등 순차적 지시사항
  - 각 단계별 소요 시간 및 팁
  - 재료 준비(미리 준비할 것), 양념, 조리 온도 등 상세 정보

- **AI 쿠킹 어시스턴트 챗봇:** 요리 진행 중 실시간으로 질문에 답하고 문제 해결
  - 예시 질문: "소스가 묽어요", "A 재료 대신 B 재료 써도 되나요?", "이 단계에서 불이 너무 세요"
  - 즉각적인 문제 해결책 제시
  - 재료 대체 가능성 안내
  - 조리 시간 조정 팁

### (4) 커뮤니티

사용자 간 경험 공유 및 피드백 플랫폼:

- **레시피 후기 작성:** 완성된 요리 사진, 별점 평가, 추천 내용 공유
- **레시피 평가 시스템:** 다른 사용자들의 리뷰 열람 및 참고
- **SNS 공유:** 완성된 요리를 소셜 미디어에 자동 공유

### 상황

### 흐름 (Step-by-Step)

#### Step 1: 재료 등록
cookmate 앱을 열어 냉장고 내부를 촬영한다.  
→ Google Gemini Vision이 '삼겹살, 김치, 양파'를 자동 인식·등록한다.

#### Step 2: 개인 프로필 확인
시스템이 '김OO'의 프로필을 확인한다.  
→ 프로필 정보: 보유 조리도구(에어프라이어), 선호(간단한 요리), 난이도(초보)

#### Step 3: AI 레시피 추천
AI 셰프가 보유 재료와 프로필을 분석해 최적 레시피를 추천한다.  
→ 추천 레시피: 
- "에어프라이어 김치 삼겹살 구이" (15분)
- "김치 삼겹살 볶음밥" (20분)
- "삼겹살 김치찜" (25분)

#### Step 4: 레시피 선택 및 조리 시작
'에어프라이어 김치 삼겹살 구이'를 선택한다.  
→ 단계별 가이드를 보며 요리를 시작한다.
- 1단계: 삼겹살과 김치를 썬다 (3분)
- 2단계: 에어프라이어에 넣고 180°C에서 12분 조리 (12분)
- 3단계: 양파를 곁들여 담아낸다 (2분)

#### Step 5: 실시간 문제 해결
요리 중 챗봇에게 "간이 너무 짠 것 같아요"라고 질문한다.  
→ AI 챗봇의 실시간 해결책:
- "남은 양파를 더 썰어 넣거나, 설탕을 반 스푼 추가하세요"
- "버터를 조금 더하면 풍미가 좋아질 거예요"

#### Step 6: 완성 및 공유
25분 만에 성공적으로 저녁 식사를 완성한다.  
→ 완성 사진과 함께 커뮤니티에 후기를 공유한다.
- 별점: ⭐⭐⭐⭐⭐ (5점)
- 후기: "정말 쉬워요! 강추합니다!"

## 5. 기대 효과

### 사용자 측면

- **냉장고 재료의 효율적 활용:** 보유 재료로 즉시 요리 가능 → 식비 절감 및 음식물 쓰레기 감소
- **요리 성공 경험 증대:** 초보자도 쉽게 따라 할 수 있는 가이드와 실시간 챗봇 지원 → 요리 성공률 향상
- **자신감 향상:** 처음 시도하는 요리도 AI의 도움으로 완성 가능 → 요리에 대한 자신감 증대
- **시간 절약:** 레시피 탐색 및 고민 시간 단축 → 바쁜 일상 속 효율성 증대

### 사회적 측면

- **환경 보호 기여:** 가정 내 음식물 쓰레기 배출 감소를 통한 지속 가능한 사회 조성
- **건강한 식생활 문화 조성:** 1인 가구 및 바쁜 직장인의 건강한 집밥 문화 확산
- **음식물 낭비 감소:** 사회 전체 음식물 쓰레기 배출량 감소에 기여

### 비즈니스 측면

- **안정적 수익 창출:** Freemium 모델(프리미엄 구독)을 통한 지속 가능한 수익 구조
  - 무료 기본 기능 vs 유료 고급 기능(영양 분석, 광고 제거, 무제한 저장)
  
- **추가 수익원 확보:** 식재료 유통사 및 주방용품 브랜드와의 전략적 제휴
  - 쿠팡, 마켓컬리 등과의 커머스 연동
  - 주방용품 브랜드 광고/제휴 수익
  - 부족 재료 자동 추천 및 구매 연결

- **사용자 확보 및 리텐션:** 초개인화 서비스로 높은 사용자 만족도 및 자발적 추천 유도

## 6. 생성 분야 및 활용 AI 도구

### 6.1 생성 분야

#### AI 기술 분야
- **컴퓨터 비전 (Computer Vision):** 냉장고 사진 및 영수증 분석을 위한 이미지 인식
- **자연어 처리 (Natural Language Processing):** 사용자 질문 이해 및 챗봇 응답 생성
- **멀티모달 (Multimodal):** 이미지와 텍스트를 함께 분석하여 통합된 이해 기반 추천

#### AI 응용 기술 분야
- **추천 시스템 (Recommendation System):** 사용자 프로필과 보유 재료에 기반한 개인화된 레시피 추천
- **생성형 AI (Generative AI):** 사용자 맞춤형 레시피 및 조리 가이드 생성
- **음성 비서 / 대화형 인터페이스 (Conversational AI):** AI 쿠킹 어시스턴트 챗봇 실시간 상호작용

### 6.2 활용 AI 도구

| 분야 | 도구 | 용도 |
|------|------|------|
| **비전/OCR** | Google Cloud Vision API (Free Tier) | 냉장고 사진 내 객체 인식 |
| | Google Gemini Vision API | 이미지 기반 재료 인식 및 분석 |
| **LLM·텍스트 생성** | Google Gemini Flash API (Free Tier) | 레시피 생성 및 챗봇 응답 생성 |
| **벡터·임베딩** | ChromaDB | 레시피 데이터 벡터화 및 유사도 기반 검색 (확장 시 FAISS로 전환) |
| **워크플로·에이전트** | LangChain (Python 라이브러리) | AI 에이전트 기반 멀티스텝 워크플로우 오케스트레이션 |
| | LangChain/TASK-SDK류 오픈소스 에이전트 프레임워크 | 사용자 질문 → 데이터 검색 → 응답 생성 파이프라인 |
| **데이터·백엔드** | Google Sheets | 사용자 프로필, 레시피, 로그 데이터 저장 |
| | Google Studio | 데이터 시각화 및 대시보드 |
| | Python Flask | RESTful API 백엔드 서버 |
| **프론트엔드** | React Native (Expo) | iOS/Android 모바일 앱 개발 |
| | Google Colab + Claude | 프로토타이핑 및 데이터 분석 |
| | GitHub | 버전 관리 및 협업 |
| **모니터링** | Next.js | 웹 대시보드 (향후 확장) |
| | Supabase | 인증 및 실시간 데이터 동기화 |
| **학습/실험** | Google Colab | AI 모델 학습 및 실험 |
| | Kaggle Notebook | 데이터 분석 및 EDA |

### 6.3 기술 아키텍처 (개념도)

```
사용자 모바일 앱 (React Native/Expo)
    ↓ (API 호출)
Python Flask 백엔드 (Render/Railway)
    ├─ /ingredients (재료 인식)
    │   └─ Google Gemini Vision API
    ├─ /recipe (레시피 생성)
    │   └─ Google Gemini Flash API + ChromaDB + LangChain
    ├─ /chat (요리 챗봇)
    │   └─ Google Gemini Flash API + LangChain
    ├─ /profile (사용자 프로필)
    │   └─ Google Sheets
    └─ /community (커뮤니티)
        └─ Google Sheets + Google Storage

데이터 저장
├─ Google Sheets (사용자 프로필, 레시피, 피드백)
├─ Google Cloud Storage (이미지 저장)
└─ ChromaDB (레시피 벡터 임베딩)

모니터링 & 분석
└─ Google Studio (대시보드)
```
